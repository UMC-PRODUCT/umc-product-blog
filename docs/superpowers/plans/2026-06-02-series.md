# Series Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development or superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Group multiple blog posts into ordered series independent of tags.

**Architecture:** Add a static `series` content collection and optional blog frontmatter reference. Series pages are generated at build time.

**Tech Stack:** Astro content collections, content helpers, content lint.

---

## Tasks

- Add `src/content/series/{id}.json` collection support.
- Add optional blog frontmatter `series: { id: string; order: number }`.
- Add `/series` and `/series/[id]` pages.
- Add series navigation to blog detail pages when a post belongs to a series.
- Extend content lint to validate known series ids and duplicate order values.

## Acceptance Criteria

- Posts without series keep rendering as before.
- Series pages list posts by `series.order`.
- Blog detail shows previous/next posts in the same series.

## Tests

- Series helper sorts by order.
- Content lint reports unknown series id and duplicate order.
